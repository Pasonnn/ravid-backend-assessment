# 01 Foundation Test Matrix

| Area | Scenario | Type | Expected Result | Command Or Evidence |
| --- | --- | --- | --- | --- |
| Happy path | Django bootstrap imports cleanly under local settings | Smoke | Project loads without configuration errors and Django reports no system-check issues for the local stack | `DJANGO_SETTINGS_MODULE=config.settings.local python manage.py check` |
| Happy path | Repo layout matches the documented foundation structure | Review | `config/`, `apps/`, and `tests/` match `docs/01-architecture/project_structure.md` | `find config apps tests -maxdepth 2 -type f | sort` |
| Validation | Missing required PostgreSQL or Redis configuration fails clearly instead of silently defaulting to the wrong runtime | Config validation | Settings raise an explicit configuration error or use a documented safe fallback | unset required `POSTGRES_*`, `REDIS_URL`, `CELERY_BROKER_URL`, or `CELERY_RESULT_BACKEND` values and run `DJANGO_SETTINGS_MODULE=config.settings.local python manage.py check` |
| Validation | Test settings stay isolated from local runtime assumptions | Smoke | `config.settings.test` boots without depending on local Docker-only values | `DJANGO_SETTINGS_MODULE=config.settings.test python manage.py check` |
| Auth | DRF baseline is installed with JWT authentication defaults but no reviewer-facing auth endpoints are added yet | Settings / smoke | `REST_FRAMEWORK` and SimpleJWT config load successfully and later auth work can build on them | settings assertion test or `DJANGO_SETTINGS_MODULE=config.settings.test python -c "import os; os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'config.settings.test'); import django; django.setup(); from django.conf import settings; print(settings.REST_FRAMEWORK['DEFAULT_AUTHENTICATION_CLASSES'])"` |
| Async | Celery app imports cleanly and is configured to use Redis-backed broker/result settings | Smoke | `config.celery` loads, autodiscovery is enabled, and the app is ready for later task modules | `DJANGO_SETTINGS_MODULE=config.settings.test python -c "import os; os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'config.settings.test'); import django; django.setup(); from config.celery import app; print(app.main)"` |
| Observability | Foundation settings do not block later JSON logging work or leak secrets during startup | Review / smoke | Logging configuration imports cleanly and startup checks do not print secret values | review `config/settings/base.py` logging section plus `DJANGO_SETTINGS_MODULE=config.settings.test python manage.py check` |
| Docker | Foundation environment variable inventory matches `docs/01-architecture/docker.md` | Doc / config review | Settings reference the same runtime variable names documented for the Docker stack | compare `config/settings/*.py` against `docs/01-architecture/docker.md` |
| Regression | Foundation does not introduce assessment endpoints prematurely | Search / review | `/api/register/`, `/api/login/`, `/api/upload-csv/`, `/api/perform-operation/`, and `/api/task-status/` remain unimplemented in this slice | `rg -n "upload-csv|perform-operation|task-status|/api/register|/api/login" config apps` |
| Regression | Baseline test run succeeds on the isolated settings module | Smoke | The project test suite boots on `config.settings.test`, giving later workstreams a stable base | `DJANGO_SETTINGS_MODULE=config.settings.test python manage.py test` |
